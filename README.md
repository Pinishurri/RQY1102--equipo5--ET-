## Descripción del problema

El Edificio El Mirador cuenta con 20 pisos y 160 departamentos. Actualmente presenta dificultades relacionadas con la gestión de gastos comunes, control de pagos, morosidad, procesos manuales y falta de información centralizada.

Entre los principales problemas identificados se encuentran:

- dificultades para registrar y controlar pagos;
- atrasos en el cobro de gastos comunes;
- poca visibilidad sobre residentes morosos;
- procesos administrativos manuales;
- falta de transparencia en la información;
- problemas de comunicación entre la administración y los residentes.

Estas situaciones pueden generar retrasos, mayores costos operativos y dificultades en la relación entre la administración y la comunidad del edificio.

## Solución propuesta

Se propone un sistema web de administración para centralizar y apoyar los principales procesos del Edificio El Mirador.

La solución considera funcionalidades como:

- gestión de usuarios, roles y permisos;
- gestión de residentes;
- gestión de unidades habitacionales;
- cálculo y gestión de gastos comunes;
- registro de pagos;
- control de morosidad;
- generación de comprobantes;
- configuración y envío de notificaciones;
- solicitudes de mantenimiento;
- gestión de reclamos;
- consultas y filtros;
- panel administrativo;
- generación de reportes.

La solución fue diseñada utilizando una arquitectura monolítica en capas y se complementa con un prototipo de alta fidelidad enfocado principalmente en el módulo **RF-04 Gestión de Residentes**.
